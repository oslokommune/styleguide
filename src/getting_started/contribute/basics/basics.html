Component

Should "this" be in Oslo style guide or in a project?

Anything that goes into Oslo style guide must be approved by the designers and should be general enough that it can be used in multiple settings.

How do we define a component?

A new component is born when a selection of other components, html elements and/or styles are put together and has to follow a set of rules. Each component is independent and must not inherit styles of other components.

Language

Everything must be written in english with the exception of example texts which can be Lorem ipsum.

Spacing

A component should never have spacing applied to the outside of the component bounds.

Font size

Avoid setting font size on inline components as they will normally get their size from the context of which they are used in. Typical inline components are links and icons.

Repositories

Oslo style guide consists of two repositories. Styleguide follows semver.org and each major version has a styleguide_devtools version that accompanies it.

styleguide
The styleguide repository is the product. It consists of all the components, styles, fonts and icons that make up Oslo style guide.

styleguide_devtools
The styleguide_devtools repository consumes the styleguide repository (just like any project would) and creates the very application you now use to browse and contribute to the styleguide repository.

Framework

assets hold fonts, icons and images
components has all of the components
general hold css styles and its documentation
getting_started documents how to consume and how to contribute
pages hold examples of how a selection of components make a page
system hold the core of the scss framework
utils hold helper scripts for mundane tasks
